# 章鱼外快小程序开发计划

## 一、项目概述

| 项目 | 内容 |
|-----|------|
| **项目名称** | 章鱼外快 |
| **核心功能** | 连接"报名者"（求职/兼职）与"发布者"（招人）的垂直服务平台 |
| **技术栈** | PHP（后端）+ 微信小程序（前端） |
| **基础框架** | 基于现有 zpwxsys 招聘系统改造 |

---

```sql
-- 群聊表
CREATE TABLE `zpwxsys_chat_group` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `jobid` int(11) NOT NULL COMMENT '岗位ID',
  `group_name` varchar(100) NOT NULL COMMENT '群名称',
  `owner_id` int(11) NOT NULL COMMENT '群主ID（发布者）',
  `max_members` int(11) DEFAULT 0 COMMENT '最大成员数（支付人数）',
  `notice` text COMMENT '群公告',
  `notice_status` tinyint(1) DEFAULT 0 COMMENT '公告审核状态',
  `status` tinyint(1) DEFAULT 1 COMMENT '1正常 0已解散',
  `expire_time` int(11) DEFAULT NULL COMMENT '到期时间',
  `createtime` int(11) DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_jobid` (`jobid`),
  KEY `idx_owner` (`owner_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 群成员表
CREATE TABLE `zpwxsys_chat_member` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `groupid` int(11) NOT NULL,
  `uid` int(11) NOT NULL,
  `is_muted` tinyint(1) DEFAULT 0 COMMENT '是否禁言',
  `createtime` int(11) DEFAULT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_group_user` (`groupid`, `uid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 群消息表
CREATE TABLE `zpwxsys_chat_message` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `groupid` int(11) NOT NULL,
  `uid` int(11) NOT NULL,
  `content` text NOT NULL,
  `type` varchar(20) DEFAULT 'text',
  `createtime` int(11) DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_groupid` (`groupid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 八、风险与注意事项

### 8.1 技术风险
| 风险 | 应对方案 |
|-----|---------|
| 微信服务号推送限制 | 使用订阅消息，引导用户订阅 |
| 群聊实时性 | 考虑使用WebSocket或轮询 |
| 敏感词过滤 | 接入第三方API或自建词库 |

### 8.2 业务风险
| 风险 | 应对方案 |
|-----|---------|
| 审核问题（ischeck机制） | 保留原有审核切换机制 |
| 支付接入 | 复用现有微信支付逻辑 |

### 8.3 注意事项
1. 所有涉及章鱼币的操作需要事务保护
2. 群聊内容需要敏感词过滤
3. 手机号展示需要脱敏
4. 发布前检查名片完整性
5. 签到状态需要实时同步
